# 开发指南

> 对应任务书：九.6 程序计算流程。
> 面向后续维护与二次开发，说明开发环境、脚本、计算流程与代码约定。

## 一、开发环境

| 工具 | 版本 / 说明 |
|------|-------------|
| Python | 3.12（建议 conda 环境 `concrete`） |
| Node.js | 20+（前端构建用） |
| 包管理 | pip（后端）/ npm（前端） |

```bash
# 后端环境
conda create -n concrete python=3.12 -y && conda activate concrete
cd backend && pip install -r requirements-dev.txt

# 前端环境
cd frontend && npm install
```

> `requirements-dev.txt` 含 `-r requirements.txt`，并附加 `pytest`、`httpx`。`pyproject.toml` 仅保留 `[tool.pytest.ini_options]`。

## 三、计算流程（后端）

`POST /api/projects/{id}/calculate` 的处理流程（[`backend/app/api/projects.py`](../backend/app/api/projects.py)）：

```
1. 按 user_id + project_id 取项目 data
2. 校验 STRUCTURE_REQUIRED（13 项）+ LOADS_REQUIRED（5 项）是否齐全
3. 派生构件输入（solvers/derive.py）：
     板   ← derive_slab_input(structure, loads)
     次梁 ← derive_beam_input(structure, loads)
     主梁 ← derive_main_beam_input(structure, loads)
4. 调用编排函数（材料常量由 materials.py 注入）：
     calculate_slab(...)      → 荷载→跨度→内力→正截面
     calculate_beam(...)      → 荷载→跨度→内力(边缘调整)→正截面(T/矩)→斜截面
     calculate_main_beam(...) → 集中力→包络(边缘调整)→正截面(T)→斜截面+吊筋
5. 结果写回 data.slab/beam/main_beam.result，置 initialized=true
6. 返回类型化结果；超筋(ValueError) → 400
```

**派生关系**（荷载自动传递）：

- 次梁间距 = 板单跨 = L2 / 板跨数；
- 次梁跨度 = L1 / 次梁跨数；
- 主梁跨度 = L2 / 主梁跨数；
- 板 / 次梁 >5 跨 → 五跨连续简化（derive 传 `min(spans, 5)` + width = 单跨 × 模型跨数）。

## 四、前端计算流程

- **参数页确认** → `useProject.calculate(page)`：先 `saveNow()` 刷新 → `POST /calculate` → 合并结果 → 自动保存。
- **级联**：前端顺序 `calculate('slab') → ('beam') → ('main_beam')`（三构件独立可算，无运行时数据依赖）。
- **计算书**：`buildReportDoc(data)` 纯函数把只读 `data`（含手改）装配为章节树，`ReportDocument.vue` 渲染。

## 五、代码约定

- **后端**：分层（api / auth / solvers / data），求解器纯函数化、每阶段 Pydantic 模型；新增需隔离的端点必须 `Depends(get_current_user)` 并按 `user_id` 存取。
- **前端**：Composition API + 模块级单例（不引 Pinia）；只用语义令牌（`var(--primary)` 等），**不硬编码颜色**；新增页面基于 `PageHeader` 扩展；前端任务必跑 `npm run build`。
- **材料常量**：后端 `materials.py` 与前端 `report/materials.ts` **双侧同步**，改动任一侧需同步另一侧。
- **测试**：新功能附单测；求解器改动跑 `test_teacher_validation.py` 防回归。
