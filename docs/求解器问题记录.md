## 坐标系约定（L₁/L₂）

楼盖平面定义：

| 方向 | 平行关系 | 代码字段 | 涉及构件 |
|---|---|---|---|
| **L₁** | 与 **次梁** 平行 | `SlabInput.length` | 次梁跨度方向；板的长方向 |
| **L₂** | 与 **主梁** 平行 | `SlabInput.width` | 主梁跨度方向；板的跨度方向(短向)；次梁间距方向 |

## 一、板求解器 (Slab) — `solvers/slab/solver.py`

### 问题 1：跨度方向用错（L₁ 当作 L₂）

- **位置**：`solver.py:72,89`（`calculate_spans`、`calculate_net_spans`）
- **当前代码**：
  ```python
  middle_span = inp.length / inp.spans          # 用了 length (L₁)
  span_unit = inp.length / inp.spans            # 用了 length (L₁)
  ```
- **正确做法**：
  ```python
  middle_span = inp.width / inp.spans           # 应用 width (L₂)
  span_unit = inp.width / inp.spans
  ```
- **原因**：板以次梁为支座，跨度沿 **L₂** 方向。`length` (L₁) 平行于次梁，不是跨度方向。
- **影响**：板跨度计算错误，同时导致 `width` 标记必需却从未使用。

### 问题 2：`width` 门禁标记必需但未使用

- **位置**：`models/slab.py:23` + `models/project.py:9`（`SLAB_REQUIRED_INPUT`）
- **说明**：`width` 被 `SLAB_REQUIRED_INPUT` 标记为必填，但因为问题1，求解器从未引用它。
- **修复后状态**：问题1修复后此问题自动消除。

## 三、主梁求解器 (Main Beam) — `solvers/main_beam/solver.py`

### 问题 4：`spans` 多余

- **位置**：`models/main_beam.py:26`
- **说明**：有默认值 `3`，但求解器硬编码为三等跨连续梁，忽略输入值。
- **影响**：如需支持非三跨主梁，需扩展。

### 问题 5：主梁自重取了全截面

- **位置**：`solver.py:22`（`calculate_main_beam_load` 参数）
- **当前计算**：
  ```python
  self_weight = 25.0 × (b/1000) × (h/1000) × s_beam
  # = 25 × 0.3 × 0.6 × 2.0 = 9.0 kN
  ```
- **正确做法**：应扣除板厚部分（仅腹板）：
  ```python
  dh = (h - slab_h) / 1000                       # 腹板高 = 0.48m
  self_weight = 25.0 × (b/1000) × dh × s_beam
  # = 25 × 0.3 × 0.48 × 2.0 = 7.2 kN
  ```
- **原因**：板厚范围内的自重已由板计算承担，主梁只承担板以下的腹板部分。

### 问题 6：次梁传主梁的恒载集中力算法错误

- **当前方法**：用连续梁剪力系数（`beta`）乘标准值计算次梁内支座反力。
- **正确做法**：用简支总荷载 `w × L`：
  ```python
  G_sec_beam = g_std_beam × span_beam
  # = 9.9038 × 6.0 = 59.42 kN
  ```
- **差异**：旧值 63.90 kN → 正确值 59.42 kN。
- **原因**：主梁计算只需次梁端部反力（总荷载），不需要连续梁系数。

### 问题 7：活载集中力算法错误

- **当前方法**：用连续梁剪力系数，得到 55.54 kN。
- **正确做法**：用从属面积法：
  ```python
  Qk = Q_live × (span_beam × s_beam)
  # = 4.0 × (6.0 × 2.0) = 48.00 kN
  ```
- **差异**：旧值 55.54 kN → 正确值 48.00 kN。
- **原因**：次梁传给主梁的集中力 = 面荷载 × 从属面积，不应使用连续梁系数。

### 问题 8：计算跨度取值错误

- **位置**：`solver.py:51-52`（`calculate_main_beam_internal_forces`）
- **当前代码**：
  ```python
  env = calc_envelope(G=G, Q=Q, l0=span)    # span=6.0m 统一用于所有截面
  ```
- **正确做法**：
  ```python
  L1 = span_axis / 1000.0                              # 6.0000m 轴线跨
  L2 = 1.05 × (span_axis - b_col) / 1000.0             # 5.9325m 调整跨
  M1 = (α_G·G + α_Q·Q) × L1                            # 跨中用 L1
  M_B = (α_G·G + α_Q·Q) × L2                           # 支座用 L2
  M2 = (α_G·G + α_Q·Q) × L1                            # 跨中用 L1
  ```
- **规则**：
  | 截面 | 用哪个 L | 取值 |
  |---|---|---|
  | 跨中（M1, M2） | L₁ = 轴线跨 | 6.0000m |
  | 支座（M_B, M_C） | L₂ = 1.05×(轴线-柱宽) | 5.9325m |
- **影响**：M2 由 97.45 错为 96.35（约 -1%），M_B 由 -206.69 错为 -203.75（约 +1.5%）。
- **验证**：用 L₁=6.0 计算 M2 得到 97.4481，与教师示例一致 ✓

### 问题 9：支座截面矩形/T形判断错误

- **位置**：`solver.py:107-108`
- **当前代码**：
  ```python
  flexure_sections = [
      ("1", forces.M1_max, True),          # 跨中用 T形
      ("B", abs(forces.M_B_min), True),     # 支座也用 T形 (但 bf 未正确传入)
      ("2", forces.M2_max, True),
  ]
  ```
- **当前 `calc_main_beam_flexure` 实现**：
  ```python
  def calc_main_beam_flexure(..., use_t_section=True):
      if use_t_section:
          bw = bf    # 用翼缘宽
      else:
          bw = b     # 用腹板宽
  ```
- **问题**：
  - 主梁课程简化规定：**全部截面按 T形（翼缘宽 bf）计算**，包括支座。
  - 但当前代码的 `calc_main_beam_flexure` 在 `use_t_section=True` 时确实用了 `bf`，**B支座应传入 bf=2000，当前传入的 bf 正确**。
  - 但在之前我们独立测试时（`run_main_beam_full.py`），我错误地用了 `b=300` 做矩形截面，导致 As_need=1766 而非 1508。
- **结论**：代码实现逻辑正确，但使用时应确认 `bf` 参数正确传入。

### 问题 10：支座边缘调整 V₀ 用错

- **位置**：`solver.py:65-66`
- **当前代码**：
  ```python
  va = env["VA"]                                            # 包络端支座剪力
  m_b_adj = env["M_B"] + (bw / 2.0) * va                   # 用了 VA
  m_c_adj = env["M_C"] + (bw / 2.0) * va                   # 用了 VA
  ```
- **正确做法**：
  ```python
  V0 = (G + Q) / 2.0                                        # 简支剪力
  m_b_adj = env["M_B"] + (bw / 2.0) * V0
  m_c_adj = env["M_C"] + (bw / 2.0) * V0
  ```
- **差异对比**：
  | 项目 | 旧值（用 VA=101.89） | 正确（用 V₀=64.03） |
  |---|---|---|
  | M_B | -203.38 | **-206.69** |
  | M_C | -129.40 | **-135.95** |
- **原因**：支座边缘调整应使用该支座处简支梁剪力 `V₀`，而非端支座包络值 `VA`。

### 问题 11（疑似）：斜截面有效高度 h0 取值

- **教师示例**：Vc=92.4 kN → 对应 `h0 = h - cover - d = 550mm`
- **代码公式**：`effective_depth(h, cover, bar_d/2)` → `h0 = h - cover - d/2 = 560mm`
- **代码值**：Vc = 94.08 kN（h0=560）
- **差异**：4 个教师示例中正截面用 h0=560（As_need 匹配），斜截面用 h0=550（Vc 匹配）。是否应统一尚不明确。
- **建议**：确认教师程序的斜截面 h0 取值。
