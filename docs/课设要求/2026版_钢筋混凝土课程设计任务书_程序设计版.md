# 水利水电专业钢筋混凝土课程设计任务书

## 整体式单向板肋形楼盖设计与辅助计算程序开发

## 一、课程设计目的

本课程设计以某电站厂房整体式单向板肋形楼盖为对象，要求学生完成楼盖结构平面布置、板、次梁、主梁的设计计算与配筋表达，并在此基础上开展课程设计辅助计算程序开发。

通过本课程设计，学生应达到以下目标：

<sub>1.</sub> 掌握整体式单向板肋形楼盖的结构布置原则、荷载传递路径和构件设计方法。

熟悉板、次梁、主梁的荷载计算、内力计算、正截面承载力计算、斜截面承载力计算和构造要求。

<sub>3.</sub> 能够独立完成结构平面布置图，并能根据计算结果进行合理的配筋表达。

<sub>4.</sub> 能够将课程设计中的部分计算流程程序化，完成至少一个构件模块的辅助计算程序或平台功能。

能够对程序计算结果进行人工复核，说明程序所采用的公式、参数、单位和适用范围。

培养小组协作、工程表达、程序设计、成果汇报和答辩说明能力。

## 二、设计基本资料

## 1. 楼盖平面参数

设计对象为某电站厂房整体式单向板肋形楼盖。各小组可在教师给定范围内选取设计参数，但不同小组之间的尺寸参数应有所区别，以便进行方案比较。

可选平面尺寸：

• L1 = 18 m / 21 m / 24 m 

• L2 = 30 m / 36 m 

其他基本参数：

• <sup>墙体厚度：</sup>370 mm

• 板搁置长度：120mm

<sub>•</sub> 次梁搁置长度：<sub>240mm</sub>

<sub>•</sub> 主梁搁置长度：<sub>240mm</sub>

• <sup>柱截面尺寸：</sup>350 mm x 350 mm

## 2. 工程条件

<sub>•</sub> 建筑位于非地震区。

<sub>•</sub> 结构安全级别为二级。

<sub>•</sub> 结构环境类别为一类。

## 3.

<sub>•</sub> 混凝土：梁、板均采用<sub>C20</sub>。

<sub>•</sub> 板中受力钢筋、梁中箍筋、构造钢筋：<sub>I</sub>级钢筋。

<sub>•</sub> 梁中纵向受力钢筋：<sub>II</sub>级钢筋。

如采用现行规范或课程指定材料强度设计值，应在计算书和程序说明中注明。

## 4. 荷载资料

<sub>•</sub> 钢筋混凝土重力密度： $2 5 \mathrm { k N } / \mathrm { m } ^ { 3 }$ 

<sub>•</sub> 水磨石面层：<sub>0.65</sub> $\mathrm { k N } / \mathrm { m } ^ { 2 }$ 

<sub>•</sub> 石灰砂浆抹面厚度： $1 5 \mathrm { m m }$ 

<sub>•</sub> 石灰砂浆重度： $1 7 \mathrm { k N } / \mathrm { m } ^ { 3 }$ 

楼面活荷载标准值： $4 \mathrm { k N } / \mathrm { m } ^ { 2 }$ 

恒载分项系数、活载分项系数、结构重要性系数等参数应按照课程要求采用，并在计算书和程序中保持一致。

## 三、分组与任务组织

本课程设计按小组完成，每组约 人，原则上 至 人。每组设组长 名，负责协调分工、进度安排、成果汇总和答辩组织。

小组内部应明确成员分工，建议包括但不限于以下方向：

<sub>1.</sub> 结构平面布置与截面初估。

<sub>2.</sub> 板设计计算与图纸表达。

<sub>3.</sub> 次梁设计计算与图纸表达。

主梁设计计算与图纸表达。

<sub>5.</sub> 计算程序或平台开发。

<sub>6.</sub> 计算书整理、程序测试、<sub>PPT</sub>制作与答辩汇报。

成员可以兼任多个方向，但提交成果时必须说明个人实际贡献。个人贡献说明应具体到工作内容，如 完成板计算模块程序 完成主梁内力复核 绘制个人平面布置图 整理计算书第几章 等，不得仅写 参与讨论 查找资料 等笼统表述。

## 四、设计内容

本课程设计包括传统结构设计和辅助计算程序开发两部分。

## 1. 结构设计内容

每组应完成以下结构设计工作：

进行板、次梁、主梁及柱网的合理布置。

<sub>2.</sub> 完成结构平面布置及构件截面尺寸初估。

完成楼板的荷载计算、内力计算、配筋计算和构造设计。

完成次梁的荷载计算、内力计算、正截面承载力计算、斜截面承载力计算和构造设计。

完成主梁的荷载计算、内力计算、正截面承载力计算、斜截面承载力计算和构造设计。

完成主要构件配筋结果汇总。

选取主梁或次梁结构，绘制抵抗弯矩图。

完成结构平面布置图。平面布置图每名学生均须单独提交一份。

## 2. 程序设计内容

每组应开发一个课程设计辅助计算程序或小型平台。程序形式不限，可采用<sub>Python</sub>、<sub>MATLAB</sub>、<sub>ExcelVBA</sub>、网页程序、桌面程序或其他教师认可的实现方式。程序最低要求为：至少完成板、次梁、主梁三类构件中一个构件模块的辅助计算。程序模块应至少具备以下基本功能：

输入主要设计参数，如跨度、截面尺寸、材料等级、荷载参数等。

<sub>2.</sub> 自动完成所选构件的荷载计算。

自动完成所选构件主要控制截面的内力计算。

自动完成所选构件的配筋面积计算或配筋校核。

输出清晰的计算结果，包括主要中间过程和最终结果。

<sub>6.</sub> 提供程序使用说明和测试算例。

能够与计算书中的手算结果进行对比，并说明误差或差异来源。

程序不得只做简单公式展示或静态文本说明，必须具有可运行的输入、计算和输出过程。

## 五、程序开发分层要求

为鼓励不同基础学生完成有层次的成果，程序要求分为基本要求、提高要求和加分要求。

## 1. 基本要求

达到基本要求可作为程序部分合格的依据：

<sub>1.</sub> 至少实现板、次梁、主梁中的一个构件计算模块。

程序能够接收用户输入或从参数表读取输入。

程序能够输出荷载、内力、配筋面积等主要结果。

<sub>4.</sub> 程序计算公式与计算书保持一致。

<sub>5.</sub> 程序结果经过至少一个算例复核。

程序文件结构清楚，能够正常运行。

## 2. 提高要求

达到以下要求可提高程序部分评价：

覆盖板、次梁、主梁中的两个或三个构件模块。

<sub>2.</sub> 实现板到次梁、次梁到主梁的荷载自动传递。

<sub>3.</sub> 实现截面尺寸初估和合理性校核。

实现配筋方案推荐和超配率提示。

<sub>5.</sub> 实现弯矩图、剪力图或计算结果表格自动生成。

实现计算结果导出，如 、 、 或图片。

具有清晰友好的界面，如网页界面、图形界面或交互式表格。

## 3. 加分要求

以下内容列为加分项，不作为最低完成要求：

自动生成不同活载布置下的最不利荷载组合或最不利内力结果。

<sub>2.</sub> 自动生成弯矩包络图。

自动生成抵抗弯矩图。

自动完成板、次梁、主梁全流程计算。

<sub>5.</sub> 自动生成完整或半自动计算书。

实现智能校核和错误提示，如单位错误、截面尺寸不合理、配筋不足、构造遗漏等。

在现有课程设计辅助平台基础上提出明确改进，并实现可演示功能。

加分项应以实际可运行功能为依据，答辩时应进行现场演示或提供可复核的运行结果。

严禁提交无法运行、来源不明或与本组计算结果不一致的程序。
